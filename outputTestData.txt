1360:	(or "address") so that the programmer can refer to it by that name. The "type" refers to the 
1361:	"kind" of data that you are storing (e.g. text, numbers, true/false, etc.). The type must be 
1362:	specified at the time you write a line of code to "create" (or "declare") the variable. A

1380:	types of values are written in ALL_CAPS with the underscore value used to seperate words in

1519:	3.5 Compound Boolean Expressions
1520:	...
1521:	The process of "nesting" means to place one conditional statement inside the "block" of 
1522:	another conditional statement. The logical operators: "and", "or", and "not" are 
1523:	represented with the symbols: "&&" (and), "||" (or), and "!" (not). The operator AND 
1524:	is used to combine logical boolean expressions such that ALL expressions need to be true for 
1525:	the entire statement to be true. The operator OR is used to combine logical boolean 
1526:	expressions such that ANY expression can be true for the entire statement to be true. The 
1527:	operator NOT reverses the logic so that true statements are evaluated as false (and visa 
1528:	versa). If the statement contains multiple Boolean expressions and the result of the 
1529:	evaluation of one expression has already determined the expressions result, then the 
1530:	rest of the expressions do not need to be evaluated - this is known as a "short-circuited 
1531:	evaluation". For example:
1532:	...

1534:	int    age = 17;
1535:	double iq  = 1.0;
1536:	if (age >= 18 && iq > 90) {
1537:	    // with "and" ("&&") BOTH have to be true
1538:	}
1539:	else if (age >= 18 || iq > 0) {
1540:	    // with "or" ("||") EITHER can be true
1541:	}
1542:	else {
1543:	    // Case if neither evaluates
1544:	    if (age > 0) {
1545:	        // A "nested if statement"
1546:	    }

1578:	4.1 while Loops
1579:	...
1580:	The word "Iteration" is another word for "Loop" or "Looping" which is a control 
1581:	struture to have code statements "repeat". Like conditional ("if") statements, iteration 
1582:	statements control the flow of a program's logic. Also like if statements, iteration 
1583:	statements are controlled by a Boolean expression (true or false). The result of the Boolean 
1584:	expression will result in the code executing zero or mulitple times (until the expression 
1585:	evaluates to false). The expression is evaluated each time through an iteration of the loop. 
1586:	A loop is called an "infinite" loop if the Boolean expression never evaluates to false 
1587:	(thus the loop never stops looping - or never stops running). It is possible a loop will 
1588:	never run if it evaluates to false before the body (block) of the loop. If you use the 
1589:	keyword "return" inside a loop body (block) then the loop will terminate (end) early and 
1590:	potentially exit any method the loop is written inside. The "while" loop is a event loop or 
1591:	non-deterministic loop, which means it is not usually used for counting but when you do 
1592:	not know how many times the loop needs to run (a certain event can cause the loop to 
1593:	end). Loops are used for a variety of reasons such as: repeating instructions, travelling 
1594:	(traversing) arrays and lists, and searching and sorting values. For example:
1595:	...
1596:	Example of a simple while loop
1597:	int counter = 0;
1598:	while (counter < 10) {
1599:	    counter++;
1600:	}

1609:	to end the loop, then the change value increasing (increment) or decreasing (decrement) 
1610:	that value. A "for" loop can be rewritten as a "while" loop and vice versa. A "off 
1611:	by one" error occurs when a loop runs one too many or one too few times. For Example:
1612:	...

1614:	for (int i = 0; i < 10; i++) {
